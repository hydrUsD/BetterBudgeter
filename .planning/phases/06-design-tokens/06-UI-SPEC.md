## Spacing Scale

Source: DESIGN_SYSTEM.md §4.1 (TOKEN-05)

All spacing tokens are on a 4px grid. The scale uses named steps (not every 4px increment).
Note: Step 7 (`--bb-space-7`) and step 9 (`--bb-space-9`) and step 11 (`--bb-space-11`) are intentionally skipped — not in spec.

| Token | Value | Tailwind Mapping | Usage |
|-------|-------|-----------------|-------|
| `--bb-space-1` | `4px` | `--spacing-bb-1: var(--bb-space-1)` → `p-bb-1`, `m-bb-1` | Icon-to-text gap, tight internal spacing |
| `--bb-space-2` | `8px` | `--spacing-bb-2: var(--bb-space-2)` → `p-bb-2`, `m-bb-2` | Default internal padding |
| `--bb-space-3` | `12px` | `--spacing-bb-3: var(--bb-space-3)` → `p-bb-3`, `m-bb-3` | Between related elements |
| `--bb-space-4` | `16px` | `--spacing-bb-4: var(--bb-space-4)` → `p-bb-4`, `m-bb-4` | Standard gap between components / card gaps |
| `--bb-space-5` | `20px` | `--spacing-bb-5: var(--bb-space-5)` → `p-bb-5`, `m-bb-5` | Card internal padding |
| `--bb-space-6` | `24px` | `--spacing-bb-6: var(--bb-space-6)` → `p-bb-6`, `m-bb-6` | Between sections within a page |
| `--bb-space-8` | `32px` | `--spacing-bb-8: var(--bb-space-8)` → `p-bb-8`, `m-bb-8` | Between major page sections / section gaps |
| `--bb-space-10` | `40px` | `--spacing-bb-10: var(--bb-space-10)` → `p-bb-10`, `m-bb-10` | Page-level vertical rhythm |
| `--bb-space-12` | `48px` | `--spacing-bb-12: var(--bb-space-12)` → `p-bb-12`, `m-bb-12` | Large section separation |

Exceptions: `--bb-space-7`, `--bb-space-9`, `--bb-space-11` are not defined — gaps are intentional.
Touch targets (44px minimum) are addressed by COMP-08 in Phase 10, not by a spacing token.

## Typography

Source: DESIGN_SYSTEM.md §3.1, §3.2 (TOKEN-04, TOKEN-07) + CONTEXT.md D-05, D-06, D-07

### Font Stacks

```css
--bb-font-sans: "Lucida Sans Unicode", "Lucida Grande", "Lucida Sans", Verdana,
  -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

--bb-font-mono: ui-monospace, "SF Mono", "Cascadia Code", Menlo, monospace;
```

Note: The sans stack is a **prototype**. Lucida-led for ADHD-legible open letterforms;
Verdana fallback for screen legibility. Finalize in Phase 11.

### Type Scale (TOKEN-07)

4 sizes, 2 weights only.

| Token | Size | Line Height | Weight | Usage |
|-------|------|-------------|--------|-------|
| `--bb-text-sm` | `14px` | `1.5` | `400` | Secondary text, labels |
| `--bb-text-base` | `16px` | `1.6` | `400` | Body text (minimum per TOKEN-04) |
| `--bb-text-xl` | `22px` | `1.4` | `700` | Section headings |
| `--bb-text-3xl` | `36px` | `1.2` | `700` | Hero numbers (balance display) |

### Tailwind Typographic Mappings

Exposed via the BB `@theme inline` block:

| CSS Variable | @theme inline mapping | Tailwind class |
|---|---|---|
| `--bb-text-sm` | `--text-bb-sm: var(--bb-text-sm)` | `text-bb-sm` |
| `--bb-text-base` | `--text-bb-base: var(--bb-text-base)` | `text-bb-base` |
| `--bb-text-xl` | `--text-bb-xl: var(--bb-text-xl)` | `text-bb-xl` |
| `--bb-text-3xl` | `--text-bb-3xl: var(--bb-text-3xl)` | `text-bb-3xl` |

### Base Body Style (TOKEN-04)

Applied by extending the existing `@layer base body {}` rule:

```css
@layer base {
  body {
    @apply bg-background text-foreground;
    font-size: 16px;
    line-height: 1.5;
  }
}
```

16px is the browser default — no visual regression. 1.5× line-height is a modest improvement
for readability throughout the app.

### Deferred Typography Rules (CONTEXT.md D-08)

These typographic rules are NOT implemented as global CSS in Phase 6:
- No-italics enforcement (`em { font-style: normal }`)
- Always left-aligned body text
- Never justified text
- Max 65–75 chars/line
- Headings followed by 8px spacing

Reason: global resets would touch legacy OopsBudgeter pages. Implement as component-level
conventions in Phase 10+.

## Color

Source: DESIGN_SYSTEM.md §2.1 (TOKEN-01, TOKEN-02, TOKEN-03)

### Foundation Colors (TOKEN-01)

60/30/10 split maps as follows:
- **Dominant (60%) — page surfaces:** `--bb-bg`, `--bb-surface`
- **Secondary (30%) — raised cards, interactive surfaces:** `--bb-surface-raised`, `--bb-border`
- **Accent (10%) — semantic signals:** `--bb-info` (primary accent), semantic colors for specific states

| Token | Light Mode | Dark Mode | Usage |
|-------|-----------|-----------|-------|
| `--bb-bg` | `oklch(0.985 0.002 280)` | `oklch(0.155 0.005 280)` | Page background (dominant) |
| `--bb-surface` | `oklch(1 0 0)` | `oklch(0.195 0.005 280)` | Card / container background (dominant) |
| `--bb-surface-raised` | `oklch(0.975 0.002 280)` | `oklch(0.235 0.006 280)` | Elevated surface: hover states, active cards (secondary) |
| `--bb-text` | `oklch(0.205 0.006 280)` | `oklch(0.965 0.002 280)` | Primary text |
| `--bb-text-secondary` | `oklch(0.48 0.012 280)` | `oklch(0.66 0.012 280)` | Secondary/muted text, inactive tab labels |
| `--bb-border` | `oklch(0.91 0.004 280)` | `oklch(0.28 0.006 280)` | Borders and dividers (secondary) |

### Semantic Financial Colors (TOKEN-02)

`--bb-negative` is soft coral — NOT bright red. Avoids anxiety/shame response.

| Token | Light Mode | Dark Mode | Usage |
|-------|-----------|-----------|-------|
| `--bb-positive` | `oklch(0.62 0.14 155)` | `oklch(0.72 0.14 155)` | Income, on-track, success (soft green) |
| `--bb-positive-bg` | `oklch(0.95 0.04 155)` | `oklch(0.22 0.04 155)` | Positive background tint |
| `--bb-caution` | `oklch(0.72 0.14 75)` | `oklch(0.78 0.14 75)` | Warning, approaching limit (muted amber) |
| `--bb-caution-bg` | `oklch(0.95 0.04 75)` | `oklch(0.22 0.04 75)` | Caution background tint |
| `--bb-negative` | `oklch(0.58 0.16 25)` | `oklch(0.68 0.18 25)` | Expense, over-budget (soft coral — NEVER bright red) |
| `--bb-negative-bg` | `oklch(0.95 0.04 25)` | `oklch(0.22 0.04 25)` | Negative background tint |
| `--bb-info` | `oklch(0.60 0.12 245)` | `oklch(0.70 0.12 245)` | Informational, primary accent (soft blue) |
| `--bb-info-bg` | `oklch(0.95 0.03 245)` | `oklch(0.22 0.03 245)` | Info background tint |

Accent (`--bb-info`) is reserved for:
- Active tab indicator in bottom tab bar (Phase 7)
- Primary action buttons (Phase 10)
- Informational highlights and callouts (Phase 10)
- Active state indicators

### Tailwind Color Mappings (TOKEN-01/02/03 + CONTEXT.md D-01/D-02)

All `--bb-*` color tokens are exposed as Tailwind utilities via the BB `@theme inline` block.
Naming follows shadcn convention: `--color-bb-{token-name}: var(--bb-{token-name})`.

Examples: `bg-bb-bg`, `bg-bb-surface`, `text-bb-text`, `text-bb-text-secondary`,
`bg-bb-positive-bg`, `text-bb-negative`, `border-bb-border`, etc.

## Copywriting Contract

Phase 6 is a pure CSS phase with no user-facing UI. There are no CTAs, empty states,
error states, or destructive actions in this phase. Copywriting is fully deferred to
Phase 11 (COPY-01, COPY-02) where the complete string audit occurs.

The copywriting principles that govern future phases (set here as contract defaults):

| Principle | Rule |
|-----------|------|
| No shame language | Never "you overspent," "budget failed," "watch your spending" |
| Neutral framing | "Your Food spending reached €200" not "You exceeded your budget" |
| Lead with action | Tell the user what they can do, not just what happened |
| CTA pattern | Specific verb + noun: "Sync Transactions," "Set Up Budgets," "Link a Bank Account" |
| Empty state pattern | Positive framing: "Start by..." not "You have no..." |
| Error state pattern | Problem + what to do: "Couldn't connect. Try again?" |
| Confirmation pattern | Supportive: "Remove this budget? You can recreate it anytime." not "Are you sure?" |
