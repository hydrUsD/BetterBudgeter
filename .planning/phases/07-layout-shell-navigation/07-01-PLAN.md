<objective>
Build the two BB layout primitives — `PageShell` and `PageHeader` — using the TDD RED → GREEN → REFACTOR cycle. Both are pure presentational server components that downstream BB pages (Home, Budgets, Transactions, Settings) will consume. They have no business logic; the TDD discipline locks the public contract (props, landmarks, conditional render, class application) before implementation.

Purpose: Without these primitives in Wave 1, the BB layout (Wave 3) has nothing to wrap pages with. Building them first via TDD means downstream tasks can consume a contract verified by tests rather than a hand-rolled component someone might tweak silently later.

Output:
- `src/components/layout/PageShell.tsx` — single `<main>` wrapper with max-width 768px and TabBar bottom clearance (per UI-SPEC §PageShell)
- `src/components/layout/PageHeader.tsx` — `<header>` block with `<h1>` title and optional `<p>` subtitle (per UI-SPEC §PageHeader)
- `tests/components/PageShell.test.tsx` — vitest unit tests asserting NAV-05 behaviors
- `tests/components/PageHeader.test.tsx` — vitest unit tests asserting NAV-05 behaviors

This plan covers requirement **NAV-05** (shared layout components: PageShell + PageHeader).
</objective>
