## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Tampering (Open Redirect) | TabBar href values | mitigate | All 4 hrefs are hard-coded string literals in the TABS const. No user-controlled redirect input. Per RESEARCH §Security Domain — "TabBar href values are hard-coded constants in TABS. No user-controlled redirects." |
| T-07-04 | Information Disclosure | TabBar render | accept | TabBar renders only static labels (Home, Budgets, Transactions, Settings) and structural icons. No PII, no env vars, no error messages. |
| T-07-05 | Elevation of Privilege | Active-tab detection | accept | `usePathname()` reads only the current URL pathname (already exposed by the browser). No auth state or role information is read. The active tab is a visual indicator, not an access control decision. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (`lucide-react`, `next/link`, `next/navigation` already in package.json per RESEARCH §Package Legitimacy Audit). Slopcheck not required. |
