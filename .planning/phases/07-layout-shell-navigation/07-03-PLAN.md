<objective>
Restructure the App Router so that legacy OopsBudgeter routes live in the `(legacy)/` route group with their full chrome (PasscodeWrapper, AppProvider, BudgetProvider, PageLayout, Logo, Settings, Achievements, ThemeToggle, GoToTop) intact, while the root layout shrinks to truly-global concerns only (html, body, fonts, theme, toaster, viewport). Standalone pages `/login` and `/link-bank` consequently inherit only the slim root — no tab bar, no legacy chrome.

This plan implements user decisions **D-01** (two route groups; `(legacy)/` half of the split — `(bb)/` half is Plan 04), **D-02** (standalone pages outside both groups), **D-03** (5 legacy `git mv` paths), **D-04** (`(legacy)/layout.tsx` full chrome stack with exact nesting), **D-05** (slim root `layout.tsx`), **D-13** (no middleware changes), and **D-14** (`/dashboard` 308 redirect preserved after move).

Purpose: This is the structural foundation that lets Plan 04 add the BB chrome (`(bb)/layout.tsx` with TabBar + PageShell per D-06) without imposing it on legacy or standalone pages. Doing the legacy relocation first (Plan 03) and the BB wiring second (Plan 04) keeps each plan focused on one trust boundary and lets us verify NAV-04 (legacy preservation) and NAV-06 (standalone pages) before introducing the BB-specific layout.

Output:
- Slimmed `src/app/layout.tsx` (chrome removed per D-05, `viewportFit: "cover"` added per RESEARCH Pitfall 1)
- New `src/app/(legacy)/layout.tsx` containing the legacy chrome stack verbatim per D-04
- 5 `git mv` operations per D-03 relocating `legacy/`, `analytics/`, `achievements/`, `dashboard/`, `legacy-index/` into `(legacy)/`
- One human-verify checkpoint after the moves to smoke-test all 5 legacy URLs + `/login` + `/link-bank` in the browser (verifies D-02, D-14)

This plan covers requirements **NAV-04** (legacy routes accessible without tab bar) and **NAV-06** (standalone pages render without tab bar — D-02). The verification of NAV-06 is partial here (we confirm `/login` and `/link-bank` render without legacy chrome per D-02 — Plan 04's TabBar addition will complete the NAV-06 picture by confirming the tab bar is also absent).
</objective>
