## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Tailwind v4 source-scan removal | Removing `source("../../src")` from globals.css makes Tailwind scan `.planning/` + `docs/` and generate invalid CSS from class-shaped shorthand. `next dev` crashes; `bun run build` tolerates it. | advisory | Fix is committed (`59e8204`) with explanatory comment at the top of globals.css. Memory at `~/.claude/projects/.../memory/tailwind-source-restriction.md` documents the v4.1+ upgrade path to the cleaner `@source not` directive. |
| Worktree base drift | Executor worktrees can spawn from old commits (08-05 forked at `5e787a3` instead of `dbe83887`). | advisory | The `<worktree_branch_check>` block in executor prompts asserts `git merge-base` against `EXPECTED_BASE` and resets via `git reset --hard` if drift detected. Always include this block when dispatching executors. |
| Naive substring assertions on source files | `expect(content).not.toContain("use client")` matches comment text. | advisory | Use line-anchored regex like `/^["']use client["']/m` instead. Caught in 08-04. |
| Double-clamping `BudgetProgress.remainingAmount` | `calculateAllBudgetProgress()` already applies `Math.max(0, ...)`. Adding a second clamp is redundant. | advisory | Use `p.remainingAmount` directly. Documented inline in `src/lib/safe-to-spend.ts`. |
| `SyncTransactionsButton accountCount={0}` self-rendering | The button renders its OWN "Link a Bank" CTA when count=0; reusing it for an external "Link your bank" CTA produces unexpected labeling. | advisory | Use a direct `<Link href="/link-bank"><Button>` for any 0-accounts full-screen CTA. Documented in 08-RESEARCH.md Pitfall 7. |
